# MC 命令可视化编辑器 - 用户指南

> 面向浏览器的 Minecraft 指令工作台，聚焦节点编排、参数编辑、结果预览和快速导出。

## 简介

这个工具的目标不是替代 Minecraft 原生命令，而是把原本在游戏内难编辑、难发现参数、难看清逻辑关系的部分搬到一个更容易操作的工作台里。

你仍然会使用 `give`、`summon`、`execute`、`fill` 这些原生命令名词，但可以通过更稳定的桌面界面完成：

- 节点式逻辑编排
- 参数填写与候选选择
- 结果预览与问题检查
- 项目保存、导入、导出

## 快速开始

### 进入工作台

1. 打开应用首页
2. 点击 `开始创建`
3. 进入节点工作台

如果你已经有原生命令，也可以点击 `从命令导入` 进入导入弹窗。

### 完成第一条命令

1. 在左侧节点库中找到命令节点，例如 `Give`
2. 将节点添加到画布
3. 在右侧参数面板填写目标、物品、数量等参数
4. 在结果区查看生成结果和问题提示
5. 复制生成的命令，或继续扩展逻辑

## 首页结构

首页主要承担 3 件事：

- 进入节点工作台
- 从已有命令导入
- 搜索已有命令或模板

首页还会展示：

- 当前支持的高频节点类型
- 最近使用的命令
- 当前产品的核心定位

## 节点工作台

工作台分为几块主要区域：

### 左侧节点库

这里用于添加节点。当前优先支持的节点包括：

- `Give`
- `TP`
- `Summon`
- `Execute`
- `Fill`
- `Setblock`
- `Effect`
- `Particle`
- `PlaySound`

节点库会按功能分组，并支持搜索。

### 中央画布

画布用于组织逻辑结构。当前主要区分两类连线：

- 执行流：表示先后顺序
- 数据流：表示参数来源

你可以通过画布看清：

- 哪个节点先执行
- 哪个节点依赖其他节点的输出
- 哪些节点存在缺参或未接入执行流的问题

### 右侧参数面板

选中节点后，右侧会显示参数配置。

当前面板已经支持：

- 常用参数
- 高级参数
- 输入来源说明
- 资源候选
- 节点级错误提示

### 结果与问题区

结果区不仅显示生成的命令，还会显示当前图存在的问题，例如：

- 缺少必填输入
- 未接入执行流
- 节点孤立
- 类型不匹配

## 命令编辑器

经典命令编辑器适合快速查看和调整单条命令。

它包含：

- 命令类型选择
- 参数输入
- 实时预览
- NBT 编辑器

如果你从搜索结果、模板、导入命令或最近使用中进入经典编辑器，系统会进入这条工作流。

## 搜索与模板

搜索面板支持查找：

- 命令
- 模板
- 搜索历史

搜索可以按以下维度匹配：

- 名称
- 描述
- 标签

当前支持的标签包括：

- 基础命令
- 方块操作
- 实体操作
- 条件执行
- 数据操作
- 游戏设置
- 消息与反馈
- 世界操作

模板适合作为快速起点，例如：

- 给予钻石
- 召唤僵尸
- 传送到出生点
- 条件执行

## 导入命令

导入弹窗支持单行或多行导入。

你可以：

- 直接粘贴原生命令
- 从剪贴板读取
- 跳过空行
- 移除注释
- 将导入记录写入历史

多行导入时，每行一条命令。

## 项目与保存

项目设置页当前支持：

- 项目名称
- 项目描述
- 目标 Minecraft 版本
- 自动保存
- 存储前缀
- 导入导出项目 JSON

项目模型已经支持主工作流和多个子工作流，为后续函数节点和封装能力做准备。

## 资源输入

资源相关参数不必完全手写。

当前系统已经开始支持资源候选，例如：

- 物品
- 方块
- 实体类型
- 其他资源型参数

简单场景可以直接在参数面板里选择，复杂场景后续可进一步通过输入节点表达依赖关系。

## 常见问题

### 为什么有些节点会报错？

常见原因有：

- 必填参数没有填写
- 节点没有接入执行流
- 数据类型不匹配
- 节点没有接入任何工作流

### 为什么我改了参数但图里结果没变？

先确认：

- 当前选中的是否是正确节点
- 参数是否被其他节点输入覆盖
- 当前图是否存在检查器报错

### 这个工具是否主要面向移动端？

不是。当前版本优先围绕桌面端体验设计，移动端只保证基础可访问，不作为主要工作场景。
