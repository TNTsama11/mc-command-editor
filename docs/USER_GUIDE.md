## 简介

MC Command Editor 是一个现代化的 Minecraft 命令可视化编辑工具，旨在帮助玩家和地图制作者更轻松地创建复杂的 Minecraft 命令。

### 主要特性

- **可视化命令编辑** - 无需记忆复杂语法，通过界面选择和输入参数
- **实时命令预览** - 编辑时即时查看生成的命令
- **NBT 数据编辑** - 支持 JSON 格式的 NBT 数据编辑和验证
- **命令方块链管理** - 拖拽式排序，可视化连接线
- **数据包生成** - 一键导出完整的数据包 ZIP 文件
- **PWA 支持** - 可安装到桌面，支持离线使用
- **深色/浅色主题** - 保护眼睛，适应不同环境

## 快速开始

### 安装应用

#### 作为网页使用

直接访问 [MC Command Editor](https://your-github-username.github.io/mc-command-editor/) 即可使用。

#### 作为 PWA 安装

1. 在支持的浏览器（Chrome、Edge、Safari）中打开应用
2. 在地址栏右侧点击"安装"图标，或通过浏览器菜单选择"安装应用"
3. 确认安装

安装后，您可以：
- 从桌面/开始菜单快速启动
- 离线使用基本功能
- 获得类似原生应用的体验

### 创建第一个命令

1. 点击 **"新建命令"** 按钮
2. 在左侧面板选择命令类型（如 `give`）
3. 填写参数（目标、物品、数量等）
4. 在中间预览区查看生成的命令
5. 点击 **"复制"** 按钮复制命令到剪贴板
6. 在 Minecraft 中粘贴使用

## 命令编辑器

### 支持的命令类型

| 命令 | 描述 | 示例 |
|------|------|------|
| `/give` | 给予玩家物品 | `/give @p diamond 64` |
| `/summon` | 生成实体 | `/summon zombie ~ ~ ~` |
| `/tp` | 传送实体 | `/tp @p ~ ~10 ~` |
| `/fill` | 填充方块 | `/fill ~ ~ ~ ~10 ~10 ~10 stone` |
| `/setblock` | 设置单个方块 | `/setblock ~ ~ ~ diamond_block` |
| `/kill` | 清除实体 | `/kill @e[type=zombie]` |
| `/clear` | 清除物品 | `/clear @p diamond` |
| `/execute` | 条件执行 | `/execute at @p run say hello` |

### 参数类型

| 类型 | 描述 | 示例 |
|------|------|------|
| 目标选择器 | 选择实体 | `@p`, `@a`, `@e`, `@r`, `@s` |
| 位置坐标 | 3D 坐标 | `~ ~ ~`, `10 64 -20` |
| 物品 ID | 物品标识符 | `minecraft:diamond`, `diamond` |
| 数字 | 整数或小数 | `64`, `3.14` |
| NBT | 复杂数据 | `{CustomName:"Test"}` |

### 目标选择器详解

```
@p - 最近的玩家
@a - 所有玩家
@e - 所有实体
@r - 随机玩家
@s - 命令执行者
```

带参数的选择器示例：
```
@e[type=zombie,distance=..10] - 10 格内的僵尸
@a[gamemode=survival] - 生存模式的玩家
@p[limit=3,sort=nearest] - 最近的 3 个玩家
```

## 命令方块链

### 方块类型

| 类型 | 颜色 | 描述 |
|------|------|------|
| 脉冲 | 橙色 | 单次执行，需要红石或手动激活 |
| 链式 | 绿色 | 连续执行，当前置方块执行后触发 |
| 循环 | 紫色 | 持续执行，每游戏刻执行一次 |

### 条件模式

启用 **条件模式** 后，只有当前置方块成功执行时，当前方块才会执行。

### 拖拽排序

1. 将鼠标悬停在命令方块上
2. 拖拽左侧的手柄图标
3. 放置到目标位置

### 导出

支持导出为：
- **NBT 结构文件** - 可在 Minecraft 中使用结构方块加载
- **函数文件** - 用于数据包

## 数据包生成

### 创建数据包

1. 切换到 **"数据包"** 标签
2. 点击 **"新建数据包"**
3. 填写基本信息：
   - 名称（命名空间）
   - 描述
   - Minecraft 版本
4. 创建函数文件
5. 添加命令到函数
6. 点击 **"导出"** 下载 ZIP 文件

### 数据包结构

```
my_datapack.zip
├── pack.mcmeta
└── data
    └── my_namespace
        ├── functions
        │   ├── main.mcfunction
        │   └── tick.mcfunction
        └── tags
            └── functions
                └── tick.json
```

### 安装数据包

1. 将 ZIP 文件放入世界的 `datapacks` 文件夹
2. 在游戏中执行 `/reload` 命令
3. 数据包会自动加载

## 常见问题

### Q: 命令复制后粘贴到 Minecraft 不起作用？

A: 确保：
1. 命令以 `/` 开头
2. 游戏中开启了作弊
3. 有足够的权限

### Q: 数据包加载失败？

A: 检查：
1. `pack.mcmeta` 格式是否正确
2. Minecraft 版本是否兼容
3. 命名空间命名是否规范（小写字母、数字、下划线）

### Q: 应用无法离线使用？

A: 确保：
1. 至少在线访问过一次
2. PWA 已正确安装
3. 浏览器支持 Service Worker
